# Security Policy

## Reporting a Vulnerability

Please do not disclose security issues publicly before maintainers have a reasonable chance to investigate and respond.

When reporting a vulnerability, include:

- affected version or commit
- impact summary
- reproduction steps
- suggested mitigation if available

If the issue involves credentials, tokens, or database access, rotate exposed secrets before sharing logs or examples.

## Scope

Security reports are especially relevant for:

- Neo4j credential handling
- graph mutation behavior
- filesystem scanning boundaries
- dependency-related risks

## Response Expectations

Maintainers should acknowledge valid reports, assess impact, and coordinate a fix or mitigation before public disclosure.
